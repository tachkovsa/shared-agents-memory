Проблема

Знание команды утекает между сессиями

01

Агенты забывают контекст

Каждая сессия начинается с нуля. Решения, принятые вчера, теряются сегодня.

02

Знание разбросано

GitHub, доки, issues, чаты и локальные файлы — единого источника правды нет.

03

Агенты дублируют работу

Параллельные агенты решают одно и то же, не зная о решениях друг друга.

04

Данные нельзя отправлять наружу

Контекст проектов нельзя свободно передавать во внешние облака.

Решение

Общий слой памяти между агентами

Чтение и запись

Агенты пишут и читают структурированную память

ArtelMemory становится общим слоем памяти между ИИ-агентами. Каждый агент сохраняет решения и достаёт нужный контекст через единый API.

  • Структурированные записи с тегами и метаданными
  • Версионирование и пометка устаревшего
Семантический поиск

Контекст достаётся семантически через Qdrant

Векторный поиск с поддержкой RU-контекста находит релевантную память по смыслу, а не по точному совпадению. Embeddings можно генерировать локально.

  • Qdrant для векторов и payload-фильтрации
  • Локальные embedding-модели без передачи документов
Ваш контур

SQLite для метаданных, ваша инфраструктура — источник правды

Лёгкий self-hosted запуск на SQLite, Qdrant для семантической памяти, локальные embeddings для безопасного индексирования. PostgreSQL — для enterprise-профиля.

  • Один Docker Compose для старта
  • Local-only режим для регулируемых контуров
